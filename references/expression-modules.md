# Expression Modules

Use these as interchangeable building blocks. Do not paste them mechanically. The goal is to keep structure stable while reducing repetitive phrasing.

## Core Rule
- Fixed structure is allowed.
- Mechanical wording is not.
- For chapter 2, do not start by listing A/B/C/D in order unless there is a specific reason.
- First identify the percentage pattern, then choose the matching opening and explanation style.

## Anti-Pattern Rotation Rule
When multiple items share the same pattern type (e.g. 6 behavioural-split items), you MUST use different opening modules for adjacent items. Never repeat the same opening module within 4 consecutive items. If you reach the end of the module list, cycle back but change the internal paragraph structure (e.g. start with data → interpretation on one item, start with judgement → data on the next).

Use when patient feedback is predominantly positive on this dimension.

1. 从问卷结果看，患者对该维度的整体反馈较为积极，主流感受主要集中在……
2. 就该维度而言，多数患者的反馈倾向较为一致，主要认可点体现在……
3. 从反馈分布看，患者对这一问题的评价整体偏向正向，其中更集中的感受表现为……
4. 调研数据表明，患者在该维度上的体验以正向为主，较为集中的反馈指向……
5. 受访患者在本维度上的评价整体向好，多数患者的感受方向保持一致，主要体现在……
6. 根据问卷反馈，这一维度在患者群体中获得了较为一致的正向体验，集中表现在……
7. 从患者端的反馈结构看，该维度的评价呈现较明显的积极趋同，核心感受围绕……
8. 综合分析显示，患者在这一问题上的态度整体偏向认可，其中更具代表性的感受是……
9. 就反馈集中度而言，该维度的患者评价方向较为统一，正向体验主要集中在……
10. 问卷结果反映，多数患者对该维度持有较为正面的实际体验，核心认可点包括……

### Conditional Recognition（条件认可型）
Use when feedback is generally positive but contingent on conditions or support.

1. 整体来看，患者对该维度持相对认可态度，但这种认可往往建立在一定条件或持续配合基础上。
2. 调研结果反映出，多数患者认可该项表现，但在具体执行或持续坚持方面仍带有一定条件性。
3. 从反馈结构看，患者总体评价较正向，不过这种体验更多与日常管理习惯和配套支持有关。
4. 患者端的反馈虽以认可为主，但分析后可发现这种认可存在较明显的"条件依赖"特征。
5. 该维度的评价呈现出"有条件认可"的格局——患者在基础层面接受度高，但持续执行仍需更多外部支撑。
6. 调研数据显示，患者对该维度的满意度与日常支持力度高度相关，正向体验并非独立存在。
7. 从患者视角看，该维度的正向反馈并非无条件的——它依赖于持续的监测、提醒和教育配套。
8. 多数患者的认可背后隐藏着一个共同前提：需要持续的信息获取和管理支持作为基础。

### Behavioural Split（行为分化型）
Use when patient behaviour/experience is clearly split across multiple options.

1. 从本题占比结构看，患者在这一问题上的行为和感受并未明显集中于单一方向，呈现出一定差异。
2. 调研结果提示，患者在该维度上的执行方式和主观体验存在分层，尚未形成高度一致的反馈。
3. 就该问题而言，不同选项之间占比较为接近，反映出患者在实际生活场景中的执行情况并不完全相同。
4. 各选项之间的分布较为分散，患者的用药行为和体验在不同路径上均有一定比例，提示群体内部存在明显的行为分化。
5. 从占比结构观察，该问题的反馈呈现出多中心分布特征，患者的执行模式分布在多个不同的类型上。
6. 问卷结果显示，患者对该问题的选择未形成单一主流，不同行为模式和习惯路径均有相当数量的患者对应。
7. 本题的反馈分布呈现出较为均匀的多路径格局，反映出在该维度上患者的做法和习惯存在多样性。
8. 从数据看，该维度上患者的行为策略存在几种并列的主要模式，没有任何一种占据绝对主导地位。
9. 在患者群体的日常管理中，这一维度展现出几种具有代表性的行为类型，不同患者根据自身条件选择了不同的执行路径。
10. 该问题的反馈结构提示，患者在该维度上的行为并非"对与错"的二分，而是几种常见模式的并行存在。

### Pain-Point Attention（痛点关注型）
Use when there are significant pain points or concerns that need highlighting.

1. 本题结果提示，除主流正向体验外，患者在该维度上仍保留较明显的痛点或顾虑。
2. 从反馈分布看，这一问题并非单纯的正向认可结构，部分患者的关注点更多集中在……
3. 调研结果反映出，患者在评价该维度时，除认可因素外，仍明显关注使用中的不便或管理难点。
4. 从数据分布看，即便存在一定比例的正面反馈，该维度仍有显著体量的患者表达了实际困难或未满足的需求。
5. 该维度呈现出一种"认可中存在痛点"的结构特征——正向和负向体验并存，且痛点不可忽略。
6. 调研反馈提示，该问题所涉及的实际困难或管理障碍，在当前的日常用药场景中具有不容忽视的普遍性。
7. 除少数体验较好的患者外，本题中有相当比例的患者反映出在使用或管理过程中遇到的具象问题。
8. 从反馈的痛感强度看，该维度不是个别现象，而是一个在相当比例患者中反复出现的系统性关注点。
9. 该问题的分布结构提示，不是"有没有问题"的争论，而是"问题有多大范围、影响有多深"的讨论。
10. 在患者的实际反馈中，该维度的痛点并非边缘性抱怨，而是在日常管理中具有实质性影响的关键障碍。

## Chapter 2: Interpretation Modules

### Experience / Symptom（体验与症状感知）

1. 这类反馈通常反映出患者对实际身体感受和日常状态变化的直观判断。
2. 该结果在一定程度上体现了患者对用药后体验变化的主观感受。
3. 从患者视角看，更看重的往往不是抽象指标，而是身体是否舒服、症状是否减轻、生活是否更稳定。
4. 患者对用药体验的评价本质上是对"吃药后我的身体感觉好不好"这一朴素问题的综合回答。
5. 该维度的反馈反映了患者对诊疗效果的最直接感知——不是实验室数值，而是日常生活中的身心变化。
6. 在慢病场景下，患者的体验判断往往融合了身体感受、症状记忆和对"正常状态"的自我定义。

### Adherence / Behaviour（依从性与行为）

1. 这一分布更多体现患者在长期坚持用药和执行管理要求方面的真实行为特征。
2. 在慢病管理场景下，是否能持续执行往往直接影响患者对方案的整体评价。
3. 这类结果说明患者关注的不只是药物本身，也包括是否容易坚持、是否容易融入日常生活。
4. 依从性反馈实际上揭示了药物在真实世界中的"可执行性"——不仅是药理上的合理性，更是生活中的可操作性。
5. 患者的行为选择往往不是单一因素驱动的，而是习惯、记忆、便利性和认知水平的综合体现。
6. 从行为角度看，该维度的反馈反映了一个核心张力：规范管理的要求与日常生活的随机性之间的持续博弈。

### Convenience / Cost（便利性与经济负担）

1. 这类反馈通常反映患者对用药便利性和经济承受度的综合感受。
2. 对患者而言，是否容易购买、是否方便服用、价格是否可接受，都会影响长期使用意愿。
3. 从日常管理逻辑看，便利性和负担感往往是影响坚持度的重要因素。
4. 便利性不是锦上添花的附加项——对长期用药的患者而言，它直接决定了"能不能长期坚持"这个根本问题。
5. 患者在经济和便利性维度的反馈往往直接影响其对治疗方案的整体满意度和持续意愿。
6. 在现实世界场景中，获取药物的便利程度和经济压力，有时比药物本身的属性更直接地影响患者的用药行为。

### Guidance / Support（指导与支持）

1. 这一分布反映出患者对用药信息清晰度和支持服务可获得性的实际体验。
2. 对患者来说，是否容易理解说明、是否能及时获得指导，会直接影响用药信心和执行稳定性。
3. 相关反馈提示，该维度不仅影响认知，也影响患者在日常使用过程中的安心程度。
4. 患者对指导支持的需求本质上是一种"安全感需求"——知道有人可以问、有资料可以查、有疑问有出路。
5. 信息支持的质量往往决定了患者在面对用药疑问时的行为走向：是主动寻求解答，还是置之不理。
6. 在缺乏有效支持的情况下，患者通常会用"自己的理解"或"别人的经验"来填补信息空白，这可能是风险的来源。

### Bodies
1. 该结果提示，虽然整体反馈较好，但在……方面仍存在局部不足。

5. 患者在该维度上的差异提示，现有的患者教育或管理工具尚未完全覆盖实践中的多样化需求。
6. 该维度的分散分布说明，即使在整体评价较好的背景下，局部环节的规范化仍有提升余地。
7. 从数据看，这部分反馈所揭示的不是根本性缺陷，而是从"够用"到"更好"之间的持续性优化机会。
8. 患者之间行为的不一致往往指向一个共同的根因：该环节缺乏足够具体、可操作的患者指引。
9. 这些局部差异的可贵之处在于，它们为精准化患者干预提供了具体抓手，而非模糊的方向感。
10. 从改进优先级看，这些待完善点虽然影响范围不同，但在高风险或关键管理场景下的边际价值较大。

## Prompt Reinforcement
When converting these modules into prompt instructions, explicitly require:
- Start from judgement, not from enumeration
- Embed percentages naturally
- Vary opening sentences across adjacent items — do NOT repeat the same module within 4 consecutive paragraphs
- Choose expressions according to data pattern
- Avoid repeated paragraph skeletons across all questions
- Never switch into doctor-viewpoint analysis (no prescribing strategy, risk stratification, or clinical decision path)
- For introduction, vary the disease-background opening, product relevance sentence, and purpose closing instead of reusing one stock paragraph every time
- In batch scenarios (multiple reports with different product/region), vary the module rotation starting point for each report to reduce inter-report similarity
- When cycling back through modules due to high item count, change the internal paragraph structure (e.g. data → interpretation vs judgement → data) to avoid perceptible repetition
